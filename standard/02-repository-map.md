# Repository Map

## Root Files

### `README.md`

Primary user-facing documentation. Explains the purpose, setup steps, folder
structure, session workflow, skill search and approval, manual commands, and
end-to-end requirements.

### `CLAUDE.md`

Startup instructions for Claude Code. It tells the assistant to run the
briefing, search skills before tasks, honor the approval gate, and update state
after reviews.

### `.env.example`

Template for local environment configuration.

Expected values:

```env
GITHUB_TOKEN=your_github_token_here
DEFAULT_REPO=owner/your-repo-name
```

`GITHUB_TOKEN` is required for private repos and posting comments. Public repo
fetching may work without it but is rate-limited and cannot post.

### `.env`

Local secret/config file. It must not be committed. It is loaded by
`python-dotenv` from repo root.

### `.gitignore`

Ignores local secrets, generated review outputs, Python cache files, and local
Chroma data.

### `requirements.txt`

Runtime Python dependencies:

- `chromadb`: semantic skill search.
- `python-dotenv`: load `.env` from repo root.

### `requirements-dev.txt`

Developer dependency entry point. Currently includes runtime requirements only.
Use this as the future place to add linters or test-only packages.

## `agents/`

### `agents/schedule.json`

Defines scheduled review tasks. Each task has a name, skill pipeline label,
cadence, last run date, and target repo.

### `agents/review_state.json`

Stores per-repo review state and review history. Updated after completed review
runs by `scripts/update_schedule.py`.

### `agents/skills-registry.json`

Source of truth for active and pending skills. `scripts/search_skills.py`,
`scripts/index_skills.py`, and `scripts/approve_skill.py` read or update this
file.

## `scripts/`

### `scripts/common.py`

Shared filesystem and JSON helpers:

- repo-root path constants.
- `repo_slug(repo)`.
- `display_path(path)`.
- `load_json(path)`.
- `atomic_write_json(path, data)`.
- `write_text(path, content)`.
- `read_text(path)`.

### `scripts/github_client.py`

Shared GitHub API client:

- loads `.env` from repo root.
- builds standard GitHub headers.
- handles GET and POST JSON calls.
- exits with clear errors on GitHub HTTP/URL failures.
- supports paginated collection fetching.

### `scripts/briefing.py`

Prints due/upcoming scheduled review tasks and recent review history.

### `scripts/fetch_github.py`

Fetches a PR, its changed files, and commits from GitHub. Saves a normalized
diff JSON artifact to `outputs/reviews/`.

### `scripts/search_skills.py`

Finds active skills for a query. Uses Chroma when `.chroma/` exists and contains
skills. Falls back to keyword text search if Chroma is missing or fails.

### `scripts/index_skills.py`

Indexes all active skills from `agents/skills-registry.json` into Chroma.

### `scripts/create_skill.py`

Creates a pending skill markdown file and registers it under `pending`.

### `scripts/approve_skill.py`

Moves a pending skill into active skills, updates the registry, and attempts to
index it into Chroma.

### `scripts/update_schedule.py`

After a review completes, updates schedule and state JSON. It refuses unknown
repos and writes JSON atomically.

### `scripts/post_review_comment.py`

Posts a generated markdown review report to the GitHub PR as an issue comment.

## `tests/`

### `tests/test_scripts.py`

Stdlib `unittest` coverage for shared JSON writing, update-schedule failure
behavior, and create-skill output.

## Generated/Local Directories

### `.chroma/`

Local Chroma vector database for semantic skill search. Generated by
`scripts/index_skills.py`. Ignored by git.

### `outputs/reviews/`

Generated PR diff JSON and review markdown reports. Ignored by git.

### `standard/`

This context pack. It is intended to be committed because it documents how the
repo works for future maintainers and AI agents.
